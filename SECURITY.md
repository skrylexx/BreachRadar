# Security Policy

## Supported Versions

Currently, only the latest `main` branch is actively supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| v0.4.x  | :white_check_mark: |
| < 0.4.0 | :x:                |

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

If you believe you have found a security vulnerability in BreachRadar, please report it to us confidentially so we can fix it before it is exploited.

Please send an email to: **skrylexx@proton.me**

In your email, please include:
- A description of the vulnerability.
- Steps to reproduce the issue (including any payload or specific configuration).
- The potential impact (e.g., unauthorized access, data leak).

### Response Time
- We will acknowledge receipt of your vulnerability report within **48 hours**.
- We aim to triage and provide a timeline for a fix within **7 days**.
- Once the issue is resolved, we will publish a security advisory and credit you for the discovery (if you wish).

Thank you for helping keep BreachRadar safe!

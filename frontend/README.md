# BreachRadar — Frontend WebUI

This directory contains the BreachRadar user interface, built with Next.js.

## 🚀 Technologies Used

- **Framework**: [Next.js 15](https://nextjs.org/) (App Router)
- **Language**: [TypeScript](https://www.typescriptlang.org/)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/)
- **Components**: [Shadcn UI](https://ui.shadcn.com/) (based on Radix UI)
- **Charts**: [Recharts](https://recharts.org/)
- **Form Management**: React Hook Form & [Zod](https://zod.dev/)
- **Internationalization**: [next-intl](https://next-intl-docs.vercel.app/) (French / English)
- **State Management**: [Zustand](https://github.com/pmndrs/zustand)

## 🛠️ Frontend Features

1. **Monitoring Dashboard**: Global visualization of the domain's security status.
2. **Alert Management**: Interface to consult data leaks, ransomware, and CVEs.
3. **Administration**: User management, API keys, and system settings.
4. **OSINT Visualization**: Dedicated pages for each intelligence tool (RansomLook, CVE, etc.).
5. **Reporting**: Interface for viewing and downloading reports generated by the backend.

## 💻 Local Installation (without Docker)

If you want to run the frontend natively for development:

### 1. Prerequisites
- Node.js 20+ installed.
- npm or pnpm.
- The BreachRadar backend must be accessible (usually at `http://localhost:8000`).

### 2. Environment Setup
```bash
# Go to the frontend directory
cd frontend

# Install dependencies
npm install
```

### 3. Configuration
The frontend uses the `.env` file located at the project root (`../.env`).
Ensure `NEXT_PUBLIC_API_URL` is correctly set (e.g., `http://localhost:8000`).

### 4. Running
```bash
# Run the development server with Turbopack
npm run dev
```
The interface will be accessible at [http://localhost:3000](http://localhost:3000).
